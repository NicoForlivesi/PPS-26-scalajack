# Sprint 0

Lo Sprint 0 è costituito da una serie di incontri iniziali tra i tre componenti del gruppo, con lo scopo di impostare il
progetto prima dell'inizio dello sviluppo vero e proprio.

## Obiettivi

- Analizzare il dominio del Blackjack e redigere la [Requirement Specification](../report/requirement_specification.md).
- Definire il [Product Backlog](product_backlog.md) e ordinarlo per priorità.
- Concordare il [Processo di sviluppo](../report/processo_di_sviluppo.md): metodologia Scrum, ruoli, workflow Git e
  Definition of Done.
- Creare e configurare il repository su GitHub, impostando i due branch principali (`master` e `develop`) e la struttura
  del progetto `sbt`.
- Definire le principali scelte architetturali di massima, in particolare l'adozione del pattern **MVC** e
  l'organizzazione del codice per **moduli**.

## Sprint Review

Al termine dello Sprint 0 il gruppo dispone di un backlog condiviso, di un repository configurato e di una visione
architetturale comune. Si può quindi procedere con la pianificazione del primo sprint di sviluppo.
